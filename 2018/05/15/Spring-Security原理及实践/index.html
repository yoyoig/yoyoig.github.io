Spring Security原理及实践

spring security 基本原理

spring security核心就是security的过滤器链.
绿色的过滤器,用来验证用户的身份.每一个过滤器就是一种认证的方式.

UsernamePasswordAuthenticationFilter用来处理表单登录,
BasicAuthenticationFilter用来处理httpBasic登录.

用来检测,当前请求是否有当前过滤器所需要的信息.比如对UsernamePasswordAuthenticationFilter 检测请求是否是登录请求,有没有带上用户名和密码.如果有,则尝试登录,若没有则放到下一个过滤器.
如果有其他认证方式,请求会一直往后走,直到有一个认证方式进行认证.或者走到FilterSecurityInterceptor.

FilterSecurityInterceptor,根据WebSecurityConfigurerAdapter的配置,来检查当前请求是否需要权限.若果不需要权限则放行,需要权限则判断在前面是否进行了认证,未通过则抛出对应的异常

ExceptionTranslationFilter,用来处理FilterSecurityInterceptor抛出的异常.比如对于需要权限api,引导用户去登录.

绿色的过滤器,可以通过配置,来让该过滤器是否生效.比如不配置http.httpBasic(),则BasicAuthenticationFilter不会生效.

对蓝色和橙色的过滤器,是不能通过配置来让它是否生效和改变过滤器的顺序的.
