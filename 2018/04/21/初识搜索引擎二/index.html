1.精确匹配和全文搜索

精确匹配 exact value

即搜索时,必须和field字段的值完全匹配才能搜索到.
比如:搜索字段值为2018-01-01的doc,我们必须搜索2018-01-01才能搜索到.

全文搜索 full text

同样搜索字段值为2018-01-01的doc,我们搜索2018、01、2018 01 01都可以将2018-01-01搜索出来。全文搜索还有如下功能:

缩写查询: cn <-> china

格式转化: like <-> likes

大小写转化: Tom <-> tom

同义词: like <-> love

结论: 全文搜索不单是匹配一个完整的值,可以对值拆分后进行匹配.也可以匹配到缩写、时态、大小写、同义词等进行匹配

2.倒排索引

在全文搜索时，会将搜索文本进行分词和normalization处理，然后在倒排索引中查询数据。

倒排索引的建立

我们对以下数据模拟建立倒排索引

doc1：I really liked my small dogs, and I think my mom also liked them.

doc2：He never liked any dogs, so I hope that my mom will not expect me to liked him.

由上表可知,倒排索引将对每个词指向存在该词doc。并且对liked、small、dogs进行了normalization处理。

我们用mother like little dog来进行查询时。
ES同样会将我们搜索的text进行分词、normalization处理。再从倒排索引中查询。

mother like little dog -> mom like little dog.

我们可以看到doc1和doc2中都没有以上单词,但是通过normalization处理后,能查询到doc1.

3.什么是分词器

分词器:即用来切分词语以及normalization词语的工具.

由这个解释就知道,倒排索引就是由分词器对语句处理后进行建立的。
分词器包括以下内容：

character filter:

作用：在一段文本进行分词之前，先进行预处理，比如说最常见的就是，过滤html标签

tokenizer:

作用：分词处理，hello you and me –> hello, you, and, me

token filter

作用：大小写转化、同义词转化、去除停用词等

内置分词器的介绍：

standard analyzer

simple analyzer

whitespace analyzer

language analyzer (特定的语言的分词器，比如英语分词器)

3.组合多个搜索条件

假如这样一个博客:

  • 字段title必须包含elasticsearch
  • content可以包含elasticsearch也可以不包含elasticsearch
  • author_id不能为22

我们可以通过如下语法查询

query.bool为布尔查询
布尔查询包括:

  • must :结果必须满足must子句，并且参与计算分值
  • filter:结果必须满足filter,不参与计算分值
  • should:可以不满足should,满足should时,提高分值.
  • must_not:排除满足must_not的结果

每个子查询都会计算一个document针对它的相关度分数，然后bool综合所有分数，合并为一个分数，filter不会计算分数

当仅仅需要filter进行查询时,我们需要使用constant_score,否则语法不支持仅用filter

  1. match all查询所有

  2. match指定字段是否包含该文本

  3. multi match多个字段是否包含该文本

  4. range query范围过滤查询

  5. term query查询时不将查询值分词,直接到倒排索引中查询.

  6. terms query指定多个搜索词查询,不分词

  7. 校验搜索语法是否合法,并定位

6.对搜索结果进行排序

  1. 默认排序规则

    默认情况下，是按照_score降序排序的

  2. 定制排序规则

    使用sort进行排序

  3. 使用string field进行排序

    当我们对一串string进行排序时,结果可能不准确.因为分词后会有多个单词,排序时会默认使用一个单词进行排序,导致结果不准确.

解决方案:

将string field 建立两个索引,一个分词,用来搜索,一个部分词,用来排序.

7.正排索引

  1. 正排索引

    正排索引即doc values

    • 对于如下两个doc

      doc1: { “name”: “jack”, “age”: 27 }
      doc2: { “name”: “tom”, “age”: 30 }

建立正排索引:

document name age
doc1 jack 27
doc2 tom 30

  1. 正排索引的作用

    通过正排索引可以进行排序、聚合、过滤等操作

  2. doc values的存放

    doc values是被保存在磁盘上的，此时如果内存足够，os会自动将其缓存在内存中，性能很高；如果内存不足够，os会将其写入磁盘上
