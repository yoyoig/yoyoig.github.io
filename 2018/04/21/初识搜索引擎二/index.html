1.精确匹配和全文搜索

精确匹配 exact value

即搜索时,必须和field字段的值完全匹配才能搜索到.
比如:搜索字段值为

GET /_analyze
{
// 指定分词器
“analyzer”: “standard”,
// 分词的文本
“text”: “Text to analyze”
}

// 默认查询前十条
GET /_search
{
“query”: {
“match_all”: {}
}
}

// 分页
GET /_search
{
“from”: 0,
“size”: 10,
“query”: {
“match_all”: {}
}
}

{
query:{
QUERY_NAME: {
ARGUMENT: VALUE,
ARGUMENT: VALUE,…
}
}
}

示例

GET /website/article/_search
{
“query”: {
“bool”: {
“must”: [
{
“match”: {
“title”: “elasticsearch”
}
}
],
“should”: [
{
“match”: {
“content”: “elasticsearch”
}
}
],
“must_not”: [
{
“match”: {
“author_id”: 111
}
}
]
}
}
}

GET /company/employee/_search
{
“query”: {
“constant_score”: {
“filter”: {
“range”: {
“age”: {
“gte”: 30
}
}
}
}
}
}

GET /test_index/_search
{
“query”: {
“bool”: {
“must”: { “match”: { “name”: “tom” }},
“should”: [
{ “match”: { “hired”: true }},
{ “bool”: {
“must”: { “match”: { “personality”: “good” }},
“must_not”: { “match”: { “rude”: true }}
}}
],
“minimum_should_match”: 1
}
}
}

GET /_search
{
“query”: {
“match_all”: {}
}
}

GET /_search
{
“query”: {
“match”: {
“title”: “elasticsearch”
}
}
}

// 查询出test_field或者test_field1包含test的doc
GET /test_index/test_type/_search
{
“query”: {
“multi_match”: {
“query”: “test”,
“fields”: [
“test_field”,
“test_field1”
]
}
}
}

// 在query中查询时,range会计算分值,filter中则不会
GET /company/employee/_search
{
“query”: {
“range”: {
“age”: {
“gte”: 30
}
}
}
}

GET /test_index/test_type/_search
{
“query”: {
“term”: {
“test_field”: “test hello”

}
}

GET /_search
{
“query”: {
“terms”: {
“tag”: [
“search”,
“full_text”,
“nosql” ]
}
}
}

// _validate && explain
GET /test_index/test_type/_validate/query?explain
{
“query”: {
“match”: {
“test_field”: “test”
}
}

// 对无_score的filter查询,用sort指定排序规则
GET /company/employee/_search

}

PUT /website
{
“mappings”: {
“article”: {
“properties”: {
“title”: {
// 分词,用来搜索

// 不分词,用来排序
“raw”: {
“type”: “string”,
// 不使用分词器
“index”: “not_analyzed”
}
},
// 建立正排索引进行排序
“fielddata”: true
}
}
}
}

7.正排索引

  1. 正排索引

    正排索引即doc values

    • 对于如下两个doc

      doc1: { “name”: “jack”, “age”: 27 }
      doc2: { “name”: “tom”, “age”: 30 }

    建立正排索引:
    document | name | age
    —|—|—
    doc1 | jack | 27
    doc2| tom | 30

  2. 正排索引的作用

    通过正排索引可以进行排序、聚合、过滤等操作

  3. doc values的存放

    doc values是被保存在磁盘上的，此时如果内存足够，os会自动将其缓存在内存中，性能很高；如果内存不足够，os会将其写入磁盘上
