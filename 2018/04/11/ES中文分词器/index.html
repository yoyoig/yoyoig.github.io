ES中文分词器

作者 yoyoig 日期 2018-04-11
es-ik elasticsearch
ES中文分词器

中文分词器

介绍

可以将中文搜索词拆分成一个个独立的词语,而不是将中文全部拆分成一个个汉字.还可以去除词语中的停用词比如 “么,让,在,额”等这些没用意义的词语,提高搜索效率.

1、在elasticsearch中安装ik中文分词器

  1. git clone https://github.com/medcl/elasticsearch-analysis-ik
  2. git checkout tags/v5.2.0
  3. mvn package
  4. 将target/releases/elasticsearch-analysis-ik-5.2.0.zip拷贝到es/plugins/ik目录下
  5. 在es/plugins/ik下对elasticsearch-analysis-ik-5.2.0.zip进行解压缩
  6. 重启es

2、ik分词器基础知识

  • 两种analyzer，但是一般是选用ik_max_word
  • ik_max_word: 会将文本做最细粒度的拆分，比如会将“中华人民共和国国歌”拆分为“中华人民共和国,中华人民,中华,华人,人民共和国,人民,人,民,共和国,共和,和,国国,国歌”，会穷尽各种可能的组合；
  • ik_smart: 会做最粗粒度的拆分，比如会将“中华人民共和国国歌”拆分为“中华人民共和国,国歌”。

4、ik配置文件

ik配置文件地址：es/plugins/ik/config目录

  • IKAnalyzer.cfg.xml：用来配置自定义词库
  • main.dic：ik原生内置的中文词库，总共有27万多条，只要是这些单词，都会被分在一起
  • quantifier.dic：放了一些单位相关的词
  • suffix.dic：放了一些后缀
  • surname.dic：中国的姓氏
  • stopword.dic：英文停用词

踩到的坑:本人用的5.5.1版本ES和ik.发现ik并没有自动将surname.dic导入到词库中,导致查询时精确率不够.同时也没有导入中文停用词.
我们需要自己手动将surname.dic以及extra_stopword.dic配置到IKAnalyzer.cfg.xml中.

ik原生最重要的两个配置文件
  • main.dic：包含了原生的中文词语，会按照这个里面的词语去分词
  • stopword.dic：包含了英文的停用词

停用词，stopword
a the and at but.
像停用词，会在分词的时候，直接被干掉，不会建立在倒排索引中

自定义词库
  1. 自己建立词库：每年都会涌现一些特殊的流行词，网红，蓝瘦香菇，喊麦，鬼畜，一般不会在ik的原生词典里
  • 自己补充自己的最新的词语，到ik的词库里面去
  • IKAnalyzer.cfg.xml：ext_dict，custom/mydict.dic
  • 补充自己的词语，然后需要重启es，才能生效
  1. 自己建立停用词库：比如了，的，啥，么，我们可能并不想去建立索引，让人家搜索
    custom/ext_stopword.dic，已经有了常用的中文停用词，可以补充自己的停用词，然后重启es

热更新

使用自定义词库问题
  • 每次添加完，都要重启es才能生效，非常麻烦
  • es是分布式的，可能有数百个节点，你不能每次都一个一个节点上面去修改
热更新的方案
  • 修改ik分词器源码，然后手动支持从mysql中每隔一定时间，自动加载新的词库
  • 基于ik分词器原生支持的热更新方案，部署一个web服务器，提供一个http接口，通过modified和tag两个http响应头，来提供词语的热更新
热更新
  1. 下载源码

https://github.com/medcl/elasticsearch-analysis-ik/tree/v5.2.0

ik分词器，是个标准的java maven工程，直接导入eclipse就可以看到源码

  1. 修改源码

  • JDBC配置文件,放在ik文件夹的config文件下

  • 从mysql中读取热词并写入到ES内存

  • Dictionary类中主要通过该方法加载词典,我们将Mysql加载词典的方法放在该方法中

  • Dictionary类，重新加载词典方法

  • HotDictReloadThread类：就是死循环，不断调用Dictionary.getSingleton().reLoadMainDict()，去重新加载词典

  • Dictionary类中,启动线程不停的从Mysql中加载词库

  • mvn package打包代码

  • 将zip包解压到es/plugin/ik下,同时添加mysql连接jar包,重启即可

区别:自定义词库文件,需要每次重启才有效.
使用mysql,可以动态的将热词同步到es内存中.更方便管理.

遇到的坑

一.维护词典后,查不到数据

  • 在存入数据时,会将数据按照指定的分词器分词,建立倒排索引.
  • 在查询时,会将查询的数据,按照分词器词库拆分.
  • 再按照查询拆分的词,去倒排索引查询.
  • 当先有数据,之后维护词典时,会导致按照分词器拆分的词,在倒排索引中到不到数据的问题.
  • 需要重新put一下数据,重新建立倒排索引,才能解决.
